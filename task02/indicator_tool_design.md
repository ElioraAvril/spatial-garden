## 1. 产品概述

### 1.1 产品定位
一个纯前端 HTML 工具，让用户可以**选择不同股票、自由调节指标参数、即时查看重绘结果**。无需后端、无需安装，双击打开即可使用。

### 1.2 核心价值
| 痛点 | 解决方案 |
|------|----------|
| 每次改参数都要重新运行 Python 脚本 | 滑块拖动即可实时重绘 |
| 无法快速对比不同参数下的指标行为 | 参数面板始终可见，所见即所得 |
| 多只股票之间切换繁琐 | 顶部下拉框一键切换 |
| 不同指标公式不清楚参数含义 | 每个指标配 Tooltip 说明 |

### 1.3 用户场景
1. 选择"金发科技"，查看默认技术指标面板
2. 拖动 RSI 周期从 14 改为 9，观察 RSI 曲线变化
3. 调整布林带倍数从 2.0 改为 1.5，看带宽收窄后的信号
4. 切换到"宁德时代"，对比两只股票的指标形态

## 3. 模块设计

### 3.1 股票选择器
| 字段 | 说明 |
|------|------|
| 数据来源 | `outputs/{stock_id}_daily_full.csv` |
| 支持股票 | 宁德时代、比亚迪、金发科技 (可扩展) |
| 切换行为 | 自动加载对应 CSV，重置所有指标参数为默认值 |

### 3.2 指标模块清单

每个指标模块包含：**启用开关、参数控件、独立图表面板**。

#### Module A — 价格与均线 (MA)
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 均线周期 | 5, 10, 20, 60 | 四组数字输入 | 逗号分隔，至少 1 条 |
| 线色 | 系统默认 | 颜色选择器 | 每条均线可独立设色 |

输出：K线图上叠加均线。面板标题显示当前排列（多头/空头/震荡）。

#### Module B — 相对强弱指标 (RSI)
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 周期 N | 14 | 滑块 (2~50) | Wilder 平滑方式 |
| 超买线 | 70 | 数字输入 (50~95) | -- |
| 超卖线 | 30 | 数字输入 (5~50) | -- |

输出：独立 RSI 面板，含超买/超卖参考线 + 区间填色。

#### Module C — MACD
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 快线周期 | 12 | 数字输入 (2~50) | EMA 快线 |
| 慢线周期 | 26 | 数字输入 (2~100) | EMA 慢线 |
| 信号周期 | 9 | 数字输入 (2~30) | DEA 信号线 |

输出：独立 MACD 面板，柱状图 + DIF + DEA，自动标注金叉/死叉。

#### Module D — 布林带 (Bollinger Bands)
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 周期 N | 20 | 滑块 (5~60) | 中轨周期 |
| 倍数 K | 2.0 | 滑块 (1.0~4.0, 步长0.1) | 标准差倍数 |

输出：K线图上叠加三条轨线 + 半透明带宽区域。

#### Module E — 滚动波动率
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 窗口 | 20 | 滑块 (5~120) | 滚动窗口 (天) |

输出：独立波动率面板，展示年化滚动波动率曲线。

#### Module F — 滚动夏普比率
| 参数 | 默认值 | 控件 | 说明 |
|------|--------|------|------|
| 窗口 | 60 | 滑块 (20~252) | 滚动窗口 (天) |
| 无风险利率 | 1.5% | 数字输入 (0~10%) | 年化无风险利率 |

输出：独立面板，含滚动夏普曲线 + 0 参考线 + 正负区域填色。

## 4. 数据流设计

```
CSV 文件 (存量数据)
    │
    ▼
┌─────────────┐     ┌──────────────┐
│  DataLoader  │────▶│ DataFrame    │  预处理：日期解析、排序、校验
└─────────────┘     │ (内存数组)    │
                    └──────┬───────┘
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ MA Calc  │    │ RSI Calc │    │MACD Calc │  ... 每个模块独立
    └────┬─────┘    └────┬─────┘    └────┬─────┘
         │               │               │
         ▼               ▼               ▼
    ┌──────────────────────────────────────────┐
    │           Render Engine (Chart.js)        │
    │  每个指标模块 → 独立的 Chart.js instance  │
    │  参数变化 → 重新计算 → chart.update()     │
    └──────────────────────────────────────────┘
```

### 4.1 数据加载策略
- **预加载**：页面打开时自动加载默认股票（金发科技）
- **切换加载**：更换股票时 fetch CSV 文件
- **计算复用**：相同参数组合缓存结果，避免重复计算
- **纯前端计算**：所有指标计算均在浏览器端 JavaScript 完成

### 4.2 响应式更新
```
用户拖动滑块 → debounce 300ms → 重新计算指标 → chart.update()
```
- 使用 `debounce` 防止高频重绘
- `chart.update('none')` 跳过动画以提升响应速度

## 5. 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 图表库 | Chart.js 4.x | 轻量、canvas 渲染、支持 candlestick 插件 |
| CSS | 原生 CSS (无框架) | 零依赖、加载快 |
| 数据处理 | 原生 JavaScript | 指标公式简单，无需引入 math 库 |
| 数据存储 | CSV 文件 (预先计算好) | 一次计算，多次使用 |
| 布局 | CSS Grid + Flexbox | 响应式、无需额外库 |

## 7. 扩展性设计

### 7.1 新增股票
只需在配置文件 `stocks` 数组中添加一条记录，CSV 文件放在 `outputs/` 目录即可。

### 7.2 新增指标
每个指标实现为一个独立模块，包含三个函数：
```javascript
{
  id: 'rsi',
  name: 'RSI',
  defaultParams: { period: 14, overbought: 70, oversold: 30 },
  controls: [/* 参数控件定义 */],
  calculate: (data, params) => { /* 计算逻辑 */ },
  render: (chart, result) => { /* 渲染逻辑 */ }
}
```
